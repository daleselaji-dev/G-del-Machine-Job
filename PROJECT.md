# Project Manifest

## Mission

Build a resumable career agent that improves interview probability while preserving long-term career value, evidence quality and user control.

## Current objective

- Run a rolling 60-day interview campaign.
- Prioritize qualified first-round interviews and at least one second-round interview.
- Focus on large-model/enterprise-AI commercialization, product/Agent/data roles, overseas commercial roles and strong multinational bridge roles.

## Current candidate shape

- Technical-product overseas business development and channel-building experience.
- Strengths in 0→1 market development, technical translation, OSINT/customer research, negotiation and cross-functional execution.
- Main transition gaps: visible Agent portfolio, formal product artifacts, model evaluation and MaaS/API commercial vocabulary.

## Canonical files

- `SKILL.md` — executable runtime policy.
- `docs/CONSTITUTION.md` — protected goals and constraints.
- `docs/ARCHITECTURE.md` — system and information flow.
- `docs/FIELD_ONBOARDING_MAP.md` — evolving career-field map.
- `docs/EVOLUTION_PROTOCOL.md` — bounded evolution procedure.
- `docs/EVIDENCE_AND_DD.md` — evidence and due-diligence rules.
- `evals/EVALUATION_TASKS.md` — frozen regression cases.
- `memory/STATE.md` — current operating state.
- `memory/PREFERENCES_AND_COMPANY_LEDGER.md` — stable preferences and company states.
- `memory/RUN_SUMMARY.md` — compressed run history.
- `memory/VERSIONS.md` — active and previous safe versions.

## Update discipline

Daily runs update runtime memory only. Company-specific facts stay in the ledger. Policy changes require a dedicated branch/PR and frozen-evaluator tests.
